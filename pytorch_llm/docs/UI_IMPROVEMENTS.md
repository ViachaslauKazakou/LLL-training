# 🎨 Улучшения UI в Streamlit

## ✨ Новые возможности

### 1. 📊 Мониторинг обучения в реальном времени

**Отображаемые метрики:**
- **Эпоха и шаг** — текущий прогресс
- **Train Loss** — ошибка на обучающей выборке
- **Val Loss** — ошибка на валидационной выборке (главная метрика!)
- **Best Val Loss** — лучший результат за всё время

**Визуальные индикаторы:**
```
⭐ Новый рекорд!        → val_loss улучшился (меньше, чем раньше)
📊 +0.0123 от лучшего   → val_loss хуже, чем best (на сколько)
```

**Автообновление:** Метрики обновляются каждую секунду во время обучения.

---

### 2. ⏹️ Остановка обучения

**Кнопка "Остановить":**
- Доступна только во время активного обучения
- Остановка происходит после завершения текущей эпохи
- Последний checkpoint сохраняется
- Можно продолжить обучение с остановленной точки

**Использование:**
```
1. Обучение идет...
2. Нажать "⏹️ Остановить"
3. Дождаться завершения эпохи
4. Checkpoint сохранён → можно использовать или продолжить
```

---

### 3. 📋 Таблица моделей на странице генерации

**Что показывает:**
| Колонка | Описание |
|---------|----------|
| ⭐ | 🏆 — лучшая модель по val_loss |
| Модель | Имя checkpoint файла |
| Step | На каком шаге обучения создан |
| Val Loss | Validation loss (чем меньше — тем лучше!) |
| Vocab | Размер словаря (количество токенов) |
| Размер | Размер файла на диске |

**Сортировка:** Таблица автоматически отсортирована по Val Loss (лучшие сверху).

**Визуально:**
```
⭐  Модель                      Step    Val Loss  Vocab  Размер
🏆  best_model.pt               1400    2.0561    171    37.5 MB
    checkpoint_step_2000.pt     2000    2.0850    171    37.5 MB
    checkpoint_step_3000.pt     3000    2.1200    171    37.5 MB
```

---

### 4. ⭐ Установка базовой модели

**Функция "Сделать базовой":**

Копирует выбранную модель в `best_model.pt` — главную модель для inference.

**Когда использовать:**
- У вас есть checkpoint с хорошим val_loss
- Хотите использовать его по умолчанию для генерации
- Удобно для быстрого переключения между моделями

**Использование:**
```
1. Выберите модель в селекторе "Управление моделями"
2. Нажмите "⭐ Сделать базовой"
3. checkpoint_step_3000.pt → best_model.pt
4. Готово! Теперь это главная модель
```

**Защита:** Нельзя "сделать базовой" уже базовую модель.

---

### 5. 🗑️ Удаление ненужных моделей

**Функция удаления:**

Удаляет checkpoint файлы, которые больше не нужны.

**Когда использовать:**
- Освободить место на диске
- Удалить модели с плохим val_loss
- Очистить старые эксперименты

**Использование:**
```
1. Выберите модель для удаления
2. Нажмите "🗑️ Удалить"
3. Файл удаляется навсегда
4. Таблица обновляется автоматически
```

**Защита:** Нельзя удалить `best_model.pt` — главную модель!

## 🎯 Типичные сценарии использования

### Сценарий 1: Обучение и мониторинг

```
1. Вкладка "🎓 Обучение"
2. Настройка параметров: epochs=20, batch=16, patience=5
3. Нажать "▶️ Начать обучение"
4. Наблюдать за метриками:
   - Если Val Loss падает → всё хорошо! ⭐
   - Если Val Loss не меняется 5 проверок → Early Stopping
5. Дождаться завершения или остановить вручную
```

### Сценарий 2: Выбор лучшей модели

```
1. Вкладка "🎯 Генерация"
2. Посмотреть таблицу моделей
3. Найти модель с минимальным Val Loss (отмечена 🏆)
4. Если это не best_model.pt:
   - Выбрать её в "Управление моделями"
   - Нажать "⭐ Сделать базовой"
5. Теперь эта модель используется по умолчанию
```

### Сценарий 3: Очистка старых моделей

```
1. Вкладка "🎯 Генерация"
2. Посмотреть таблицу:
   - checkpoint_step_1000.pt: val_loss=3.5 (плохо)
   - checkpoint_step_2000.pt: val_loss=2.8 (средне)
   - checkpoint_step_3000.pt: val_loss=2.1 (хорошо) 🏆
3. Удалить checkpoint_step_1000.pt (плохой результат)
4. Удалить checkpoint_step_2000.pt (не нужен)
5. Оставить только лучшие модели
```

### Сценарий 4: Остановка неудачного обучения

```
1. Обучение началось: epochs=50
2. Эпоха 10: val_loss=2.5
3. Эпоха 15: val_loss=2.5 (не улучшается)
4. Эпоха 18: val_loss=2.6 (ухудшается!)
5. Решение: нажать "⏹️ Остановить"
6. Обучение остановится после текущей эпохи
7. Checkpoint сохранён, можно:
   - Изменить learning rate
   - Добавить данных
   - Начать заново
```

## 📊 Интерпретация метрик

### Val Loss — главная метрика!

```
Val Loss = 0.5   → 🎯 Отлично! Модель предсказывает почти идеально
Val Loss = 1.5   → ✅ Хорошо! Модель понимает структуру текста
Val Loss = 2.5   → 📊 Средне. Работает, но можно лучше
Val Loss = 4.0+  → ⚠️ Плохо. Модель почти не обучилась
```

### Perplexity (производная от loss)

```
Perplexity = exp(val_loss)

Val Loss = 2.0 → Perplexity = 7.4
Val Loss = 2.5 → Perplexity = 12.2
Val Loss = 3.0 → Perplexity = 20.1

Чем меньше — тем лучше!
```

### Индикаторы улучшения

```
⭐ Новый рекорд!
→ val_loss < best_val_loss
→ Модель улучшается → продолжаем обучение

📊 +0.1234 от лучшего
→ val_loss > best_val_loss на 0.1234
→ Модель не улучшается → возможно, остановить?
```

## 🛡️ Безопасность

### Защита от случайного удаления

- ❌ Нельзя удалить `best_model.pt`
- ⚠️ Удаление необратимо (файл удаляется навсегда)
- ✅ Таблица обновляется автоматически после удаления

### Защита от перезаписи

- `best_model.pt` автоматически обновляется только при улучшении val_loss
- Ручная установка базовой модели требует явного действия
- Старая `best_model.pt` перезаписывается (сделайте backup если нужно!)

## 💡 Советы

### 1. Как часто проверять val_loss?

```python
# В config.py:
eval_every = 500  # По умолчанию — каждые 500 шагов

# Рекомендации:
- Малые данные (<1MB): eval_every=100-200
- Средние (1-100MB): eval_every=500
- Большие (>100MB): eval_every=1000-2000
```

### 2. Когда останавливать обучение?

```
✅ Остановить, если:
- val_loss не улучшается 5+ проверок
- val_loss начал расти (overfitting)
- достигли нужного качества

❌ Не останавливать, если:
- val_loss всё ещё падает
- только начали обучение (< 3 эпох)
```

### 3. Сколько checkpoints хранить?

```
Рекомендации:
- Лучшие 3-5 моделей по val_loss
- Последние 2-3 checkpoint (для восстановления)
- Удалять: checkpoint с val_loss > 3.0 (если best < 2.5)

Пример:
✅ best_model.pt (val_loss=2.1)
✅ checkpoint_step_5000.pt (val_loss=2.2)
✅ checkpoint_step_8000.pt (val_loss=2.3)
🗑️ checkpoint_step_1000.pt (val_loss=4.5) → удалить
🗑️ checkpoint_step_2000.pt (val_loss=3.8) → удалить
```
