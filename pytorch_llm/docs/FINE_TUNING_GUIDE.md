# Инструкция: Дообучение модели на новых данных форума

## Сценарий
У вас уже есть обученная модель (`best_model.pt`) и новые данные с форума в JSON формате.

## Шаг 1: Загрузка и подготовка данных

### Через UI (Streamlit)

1. **Запустите приложение:**
   ```bash
   cd /Users/Viachaslau_Kazakou/Work/LLM-learn/pytorch_llm
   poetry run streamlit run app.py
   ```

2. **Откройте вкладку "📁 Подготовка данных"**

3. **Загрузите JSON файл:**
   - Нажмите на file uploader
   - Выберите ваш JSON файл (например, `forum_messages_new.json`)
   - Система покажет:
     ```
     ✅ JSON загружен и конвертирован: 50,000 символов
     
     Топиков: 45
     Сообщений: 320
     Символов: 50,000
     ```

4. **Сохраните данные:**
   - Введите имя файла: `data/forum_new.txt`
   - Нажмите "💾 Сохранить"
   - ✅ Сохранено: data/forum_new.txt

## Шаг 2: Дообучение модели

### Через UI (Streamlit)

1. **Откройте вкладку "🎓 Обучение"**

2. **Настройте режим:**
   ```
   Режим обучения: 
   ○ 🆕 С нуля
   ● 🔄 Дообучение (Fine-tuning)  ← ВЫБЕРИТЕ ЭТО
   ```

3. **Выберите checkpoint:**
   ```
   Выберите checkpoint: best_model.pt
   
   ℹ️ Информация о checkpoint:
   Global step: 1,250
   Val loss: 2.0814
   ```

4. **Укажите новые данные:**
   ```
   Новые данные для дообучения: data/forum_new.txt
   ✅ Файл найден (48.8 KB)
   ```

5. **Настройте параметры:**
   ```
   Эпохи: 10
   Batch size: 16
   Learning rate: 0.0003
   Device: auto
   ```
   
   💡 **Рекомендации для дообучения:**
   - Эпохи: 5-10 (меньше, чем при обучении с нуля)
   - Learning rate: можно оставить 3e-4 или уменьшить до 1e-4
   - Batch size: зависит от GPU памяти

6. **Запустите дообучение:**
   - Нажмите "▶️ Начать дообучение"
   - Следите за прогрессом в разделе "📊 Прогресс"

7. **Результат:**
   ```
   ✅ Обучение завершено!
   💾 Checkpoint сохранён в checkpoints/
   ```
   
   Новый `best_model.pt` будет содержать дообученную модель.

   Нажмите "✨ Сгенерировать"

4. **Оцените результат:**
   - Проверьте, появился ли стиль новых данных
   - Сравните с генерацией до дообучения

## Через CLI

### Быстрый способ

```bash
# 1. Дообучение
poetry run python train.py \
  --continue-from checkpoints/best_model.pt \
  --data data/forum_new.txt \
  --epochs 10 \
  --batch-size 16 \
  --device auto

# 2. Генерация для проверки
poetry run python inference.py \
  --checkpoint checkpoints/best_model.pt \
  --prompt "Искусственный интеллект" \
  --max-tokens 200 \
  --interactive
```

## Важные моменты

### ✅ Что происходит при дообучении:

1. **Загружается архитектура модели** из checkpoint
   - vocab_size, d_model, n_layers — всё из old модели
   - Нельзя изменить размер модели

2. **Загружаются веса (параметры)**
   - Модель уже "знает" старые данные

3. **Продолжается обучение** на новых данных
   - Модель адаптируется к новому стилю
   - Но сохраняет знания о старых данных

### ⚠️ Потенциальные проблемы:

**Проблема 1: Новый vocab**
```
Если в новых данных есть символы, которых не было в старом токенизаторе,
они будут заменены на <unk>.

Решение: Переобучите модель с нуля на объединённых данных.
```

**Проблема 2: Катастрофическое забывание**
```
При слишком долгом дообучении модель может "забыть" старые данные.

Решение: 
- Меньше эпох (5-10 вместо 30-50)
- Меньший learning rate (1e-4 вместо 3e-4)
```

**Проблема 3: Переобучение на маленьких данных**
```
Если новых данных мало, модель быстро переобучится.

Решение: 
- Минимум 100K символов новых данных
- Ранняя остановка (следите за val_loss)
```

## Мониторинг процесса

### Логи
```bash
# Смотрите логи в реальном времени
tail -f logs/training.log

# Или через grep для важных метрик
tail -f logs/training.log | grep -E "(Epoch|val_loss|checkpoint)"
```

### Метрики
- **Train loss должен снижаться** постепенно
- **Val loss** — главная метрика:
  - Снижается → всё хорошо
  - Растёт → переобучение, остановите раньше
