# 📚 OCR приложение для подготовки данных

Streamlit приложение для преобразования бумажных задачников в JSON формат для обучения модели.

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
# Основные зависимости (уже установлены)
cd /Users/Viachaslau_Kazakou/Work/LLM-learn/pytorch_llm

# Для OCR
poetry add pytesseract pillow

# Tesseract OCR (бесплатно)
brew install tesseract
brew install tesseract-lang  # Языки (русский + английский)

# Опционально: для GPT-4 Vision (платно)
# poetry add openai (уже установлен)
```

### 2. Запуск приложения

```bash
poetry run streamlit run ocr_app.py
```

Откроется браузер на http://localhost:8501

## 📋 Workflow

### Шаг 1: Загрузка сканов (Вкладка 1)
1. Нажмите "Browse files"
2. Выберите JPEG/PNG файл со сканом задачника
3. Изображение автоматически сохранится в `data_preparation/scans/`
4. Предпросмотр покажет загруженное изображение

### Шаг 2: OCR распознавание (Вкладка 2)
1. Выберите метод OCR:
   - **Tesseract** (бесплатно) — быстро, хорошо для печатного текста
   - **GPT-4 Vision** (платно, ~$0.01-0.02/изображение) — лучше для формул и рукописного текста
2. Нажмите "🚀 Распознать текст"
3. Отредактируйте распознанный текст при необходимости
4. Нажмите "➡️ Извлечь задачи"

### Шаг 3: Редактирование задач (Вкладка 3)
1. Просмотрите извлечённые задачи
2. Отредактируйте каждую задачу:
   - Вопрос
   - Решение (необязательно)
   - Ответ
   - Категория (уравнения, расчёты, и т.д.)
   - Сложность (easy/medium/hard)
   - Ключевые слова
3. Сохраните изменения (кнопка "💾 Сохранить")
4. Удалите лишние задачи (кнопка "🗑️ Удалить")
5. Добавьте новые задачи вручную (кнопка "➕ Добавить задачу")

### Шаг 4: Сохранение JSON (Вкладка 4)
1. Просмотрите предпросмотр JSON
2. Введите имя файла (или используйте автоматическое)
3. Нажмите "💾 Сохранить JSON"
4. Файл сохранится в `data_preparation/json_output/`
5. **Копируйте для обучения:**
   - Выберите сохранённый JSON
   - Нажмите "📋 Копировать в data/"
   - Файл скопируется в `data/` для обучения модели

## ⚙️ Настройки в Sidebar

### OCR методы

#### 1. Tesseract (бесплатно) ✅
- **Скорость:** Очень быстро (1-3 сек)
- **Точность:** Хорошо для печатного текста
- **Стоимость:** Бесплатно
- **Формулы:** Среднее качество
- **Рукописный текст:** Плохо

**Настройки:**
- Языки: `rus+eng` (русский + английский)
- Можно добавить другие: `rus+eng+ukr`

#### 2. GPT-4 Vision (платно) 💰
- **Скорость:** Медленно (5-10 сек)
- **Точность:** Отлично для всего
- **Стоимость:** ~$0.01-0.02 за изображение
- **Формулы:** Отличное качество (H₂O, CO₂, etc)
- **Рукописный текст:** Хорошо

**Требуется:**
- OpenAI API ключ (получите на https://platform.openai.com)

## 💡 Советы и рекомендации

### Подготовка сканов

1. **Качество изображения:**
   - Разрешение: минимум 300 DPI
   - Формат: JPEG или PNG
   - Контраст: высокий (чёрный текст на белом фоне)

2. **Улучшение качества:**
   ```python
   # Если OCR плохо распознаёт, улучшите контраст:
   from PIL import Image, ImageEnhance
   
   img = Image.open('scan.jpg')
   enhancer = ImageEnhance.Contrast(img)
   img_enhanced = enhancer.enhance(2.0)  # Увеличить контраст
   img_enhanced.save('scan_enhanced.jpg')
   ```

3. **Ориентация:**
   - Убедитесь, что текст горизонтален
   - Поверните изображение при необходимости

### Работа с формулами

- **Tesseract:** Может путать символы (O вместо 0, I вместо 1)
- **GPT-4 Vision:** Лучше понимает контекст (H₂O, а не H20)
- **Проверяйте вручную:** Всегда просматривайте химические формулы

### Оптимизация процесса

1. **Батч обработка:**
   - Сканируйте сразу много страниц
   - Обрабатывайте пакетами по 5-10 страниц

2. **Шаблоны задач:**
   - Если формат задач одинаковый, создайте шаблон
   - Парсинг будет точнее

3. **GPT-4 Vision vs Tesseract:**
   - Tesseract для простого печатного текста
   - GPT-4 Vision для формул, рукописного текста, сложных макетов

## 🐛 Устранение неполадок

### Tesseract не работает

```bash
# Проверьте установку
tesseract --version

# Если не установлен:
brew install tesseract

# Проверьте языки
tesseract --list-langs

# Если нет русского:
brew install tesseract-lang
```

### GPT-4 Vision ошибки

- **401 Unauthorized:** Неверный API ключ
- **429 Rate limit:** Превышен лимит запросов, подождите
- **413 Payload too large:** Изображение слишком большое (уменьшите до 2000x2000 px)

### Плохое распознавание

1. **Улучшите качество скана:**
   - Увеличьте контраст
   - Увеличьте разрешение (300+ DPI)
   - Выровняйте страницу

2. **Используйте GPT-4 Vision** для сложных случаев

3. **Ручное редактирование:**
   - Всегда проверяйте результат
   - Исправляйте формулы вручную

## 🔄 Workflow для большого объёма данных

### Рекомендуемый процесс:

1. **День 1: Сканирование**
   - Отсканируйте все страницы (50-100 страниц)
   - Сохраните в `scans/`

2. **День 2-3: OCR**
   - Batch обработка с Tesseract (быстро)
   - Проблемные страницы — GPT-4 Vision
   - Сохраните все OCR результаты

3. **День 4-5: Редактирование**
   - Просмотрите все задачи
   - Исправьте ошибки
   - Добавьте метаданные (категории, сложность)

4. **День 6: Финализация**
   - Объедините JSON файлы (если нужно)
   - Скопируйте в `data/`
   - Начинайте обучение!

### Скрипт для объединения JSON:

```python
# merge_json.py
import json
from pathlib import Path

def merge_json_files(input_dir, output_file):
    all_tasks = []
    
    for json_file in Path(input_dir).glob("*.json"):
        with open(json_file, 'r', encoding='utf-8') as f:
            data = json.load(f)
            all_tasks.extend(data['tasks'])
    
    merged = {
        "dataset_name": "merged_chemistry_tasks",
        "created_at": datetime.now().isoformat(),
        "total_tasks": len(all_tasks),
        "tasks": all_tasks
    }
    
    with open(output_file, 'w', encoding='utf-8') as f:
        json.dump(merged, f, ensure_ascii=False, indent=2)
    
    print(f"✅ Объединено {len(all_tasks)} задач → {output_file}")

# Использование
merge_json_files('data_preparation/json_output', 'data/all_chemistry_tasks.json')
```
