### После:

## Возможности

### 1. Автоматическое сканирование файлов
- Система автоматически находит все `.txt` и `.json` файлы в `data/`
- Файлы отсортированы по дате изменения (новые сверху)
- Показывается размер каждого файла

### 2. Визуальный выбор
- Выпадающий список вместо ручного ввода
- Видно имя файла и его размер
- Не нужно помнить точный путь

### 3. Гибкость
- Опция "✏️ Ввести путь вручную..." для файлов вне `data/`
- Можно использовать файлы из других директорий
- Сохранена возможность ручного ввода

### 4. Информативность
- Показывается полный путь после выбора: `📁 Путь: data/file.json`
- Статус файла: `✅ Файл готов к использованию (841.0 KB)`
- Предупреждение, если файл не найден: `⚠️ Файл ... не найден`

## Использование

### Сценарий 1: Выбор существующего файла

1. Откройте вкладку "🎓 Обучение"
2. Раздел "Данные для обучения" / "Новые данные для дообучения"
3. Нажмите на выпадающий список "Выберите файл данных"
4. Выберите нужный файл из списка
5. Путь автоматически подставится

**Готово!** Файл выбран, можно запускать обучение.

### Сценарий 2: Ввод пути вручную

1. Откройте вкладку "🎓 Обучение"
2. Нажмите на выпадающий список "Выберите файл данных"
3. Выберите "✏️ Ввести путь вручную..."
4. Появится текстовое поле
5. Введите путь, например: `/absolute/path/to/data.txt`

### Сценарий 3: Дообучение

1. Режим обучения: "🔄 Дообучение (Fine-tuning)"
2. Выберите checkpoint: `best_model.pt`
3. **Новые данные для дообучения:** 
   - Выберите из списка, например: `parser_1173691_20260530T171611.json (841.0 KB)`
4. Параметры: epochs=10, batch=16
5. Нажмите "▶️ Начать дообучение"

## Технические детали

### Алгоритм

```python
1. Сканирование data/ директории:
   - Поиск *.txt и *.json файлов
   - Получение размера каждого файла
   - Сортировка по дате изменения (новые → старые)

2. Формирование списка опций:
   - Формат: "filename.ext (size KB/MB)"
   - Добавление опции "✏️ Ввести путь вручную..."

3. Определение пути:
   - Если выбран файл из списка → используем путь из mapping
   - Если "Ввести вручную" → показываем text_input
   - Проверка существования файла

4. Отображение статуса:
   - ✅ Файл готов (с размером)
   - ⚠️ Файл не найден
   - 📂 Директория пуста
```

### Код

```python
# Сканирование файлов
data_dir = Path("data")
available_files = []

for ext in ["*.txt", "*.json"]:
    available_files.extend(data_dir.glob(ext))

# Сортировка по дате (новые сверху)
available_files = sorted(
    available_files, 
    key=lambda f: f.stat().st_mtime, 
    reverse=True
)

# Создание опций для selectbox
file_options = []
file_paths = {}

for f in available_files:
    size_kb = f.stat().st_size / 1024
    size_mb = size_kb / 1024
    
    size_str = f"{size_mb:.1f} MB" if size_mb >= 1 else f"{size_kb:.1f} KB"
    display_name = f"{f.name} ({size_str})"
    
    file_options.append(display_name)
    file_paths[display_name] = str(f)

# Selectbox
selected_option = st.selectbox(
    "Выберите файл данных",
    options=file_options,
    help="Файлы из директории data/"
)

# Получение пути
data_path = file_paths[selected_option]
```

## FAQ

**Q: Почему не вижу свой файл в списке?**  
A: Файл должен быть в директории `data/` и иметь расширение `.txt` или `.json`

**Q: Как использовать файл из другой директории?**  
A: Выберите "✏️ Ввести путь вручную..." и введите полный путь

**Q: Файлы отсортированы странно**  
A: Файлы сортируются по дате последнего изменения (новые сверху)

**Q: Можно ли удалить старые файлы из списка?**  
A: Да, просто удалите их из директории `data/`, и они пропадут из списка

**Q: Что если директория `data/` пуста?**  
A: Появится сообщение "📂 Директория data/ пуста" и текстовое поле для ввода пути
